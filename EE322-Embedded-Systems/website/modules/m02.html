Microprocessor & Microcontroller Architecture

Von Neumann vs Harvard Architecture

Definition — Von Neumann

A computer architecture with a single bus shared between instruction and data memory. Simple but creates the "von Neumann bottleneck" — instructions and data compete for memory bandwidth.

Definition — Harvard

A computer architecture with separate buses for instruction and data memory. Allows simultaneous instruction fetch and data access, enabling higher throughput. Used in most DSPs and ARM Cortex-M (modified Harvard).

FeatureVon NeumannHarvard
Memory busesSingle (shared)Separate
BandwidthLower (bottleneck)Higher (parallel access)
ComplexitySimplerMore complex (more pins/buses)
Self-modifying codePossibleNot possible (strict Harvard)
Examplesx86, original ARMAVR, DSPs, ARM Cortex-M*

*ARM Cortex-M uses a Modified Harvard architecture: separate I-bus and D-bus internally but a unified address space externally.

Pipelining

Concept

A pipeline overlaps the execution of multiple instructions. A classic 5-stage pipeline: IF (Instruction Fetch) → ID (Instruction Decode) → EX (Execute) → MEM (Memory Access) → WB (Write Back).

Ideal speedup with $k$ stages: $S = k$ (when the pipeline is full). In practice, hazards reduce this:

  • Data hazards: Instruction depends on result not yet written back. Solved by forwarding/bypassing or stalling.
  • Control hazards: Branch target unknown until later stages. Solved by branch prediction.
  • Structural hazards: Two instructions need the same hardware unit. Solved by resource duplication.

Branch Prediction

A 2-bit saturating counter branch predictor maintains a state machine per branch:

  • Strongly Not Taken (00) → Weakly Not Taken (01) → Weakly Taken (10) → Strongly Taken (11)
  • Two consecutive mispredictions needed to change the prediction direction.
  • Accuracy: 85–95% for typical embedded code.

The ARM Cortex-M3 pipeline (3 stages: Fetch, Decode, Execute) uses a simpler branch predictor with branch target folding to minimise the branch penalty to 1 cycle.

ARM Cortex-M Memory Map & Interrupts

Memory Map

All Cortex-M processors share a common memory map:

  • 0x00000000–0x1FFFFFFF: Code region (Flash, 512 MB)
  • 0x20000000–0x3FFFFFFF: SRAM (512 MB)
  • 0x40000000–0x5FFFFFFF: Peripherals
  • 0xE0000000–0xFFFFFFFF: System (NVIC, SysTick, debug)

NVIC (Nested Vectored Interrupt Controller)

The NVIC supports configurable priority levels (8–256 levels depending on implementation). Key features:

  • Tail-chaining: Back-to-back ISR execution without full context save/restore (6 cycles).
  • Late arrival: If a higher-priority interrupt arrives during stacking, the NVIC switches to the higher-priority ISR directly.
  • Deterministic latency: 12 cycles from interrupt assertion to first ISR instruction on Cortex-M3/M4.
