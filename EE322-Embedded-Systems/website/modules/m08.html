Analysis & Verification

Performance Profiling

Profiling identifies bottlenecks and validates real-time constraints. Techniques range from hardware-assisted to software-instrumented.

DWT Cycle Counter (Cortex-M)

// Enable DWT cycle counter
CoreDebug->DEMCR |= CoreDebug_DEMCR_TRCENA_Msk;
DWT->CYCCNT = 0;
DWT->CTRL   |= DWT_CTRL_CYCCNTENA_Msk;

uint32_t t0 = DWT->CYCCNT;
critical_function();
uint32_t cycles = DWT->CYCCNT - t0;
float us = (float)cycles / (SystemCoreClock / 1e6f);

GPIO Toggle Method

Set a GPIO pin high before the function, low after. Measure the pulse with an oscilloscope or logic analyser — zero software overhead.

Memory Analysis

Embedded systems have tight memory budgets. The linker map file shows exactly where every byte goes:

SectionContentsMemory Region
.textCode + constantsFlash
.rodataRead-only dataFlash
.dataInitialised globalsFlash → RAM (copied)
.bssZero-initialised globalsRAM

# Check firmware size
arm-none-eabi-size -A firmware.elf
# Detailed map
arm-none-eabi-nm --size-sort firmware.elf | tail -20
Stack Overflow

Each RTOS task needs its own stack. Under-sized stacks cause silent corruption. Use stack painting (fill with 0xDEADBEEF) and check high-water marks at runtime.

Failure Mode & Effects Analysis (FMEA)

FMEA systematically identifies potential failures before they occur. Each failure mode is scored:

$$\text{RPN} = \text{Severity} \times \text{Occurrence} \times \text{Detection}$$

Each factor is rated 1–10. High RPN items get priority for mitigation.

Reliability Engineering

For constant failure rate \(\lambda\) (exponential model):

$$R(t) = e^{-\lambda t}, \quad \text{MTTF} = \frac{1}{\lambda}, \quad A = \frac{\text{MTBF}}{\text{MTBF} + \text{MTTR}}$$

Triple Modular Redundancy (TMR)

Three identical modules with majority voter. System fails only if ≥ 2 modules fail:

$$R_{\text{TMR}}(t) = 3R^2(t) - 2R^3(t)$$

TMR Cross-Over

TMR is more reliable than a simplex system only when individual reliability \(R > 0.5\). Below that, redundancy increases the chance of majority failure.

FIT Rate

Failures In Time = failures per \(10^9\) device-hours. A component rated 100 FIT has \(\lambda = 10^{-7}\)/hr, MTTF = 10 million hours.

Formal Verification

Model Checking with SPIN

Express the system as a Promela model; SPIN exhaustively checks all reachable states against LTL properties.

/* Promela: producer-consumer */
mtype = { DATA, ACK };
chan ch = [2] of { mtype, byte };

active proctype Producer() {
    byte val = 0;
    do
    :: ch ! DATA, val -> val++
    od
}

active proctype Consumer() {
    byte v;
    do
    :: ch ? DATA, v -> printf("Got %d\n", v)
    od
}

LTL Properties

Express safety and liveness:

  • Safety: \(\square \neg \text{deadlock}\) — "it is always the case that there is no deadlock"
  • Liveness: \(\square(\text{request} \rightarrow \lozenge\text{grant})\) — "every request is eventually granted"

MISRA-C 2012 Guidelines

MISRA-C restricts the C language to a safer subset for safety-critical embedded code. Key rules:

RuleCategoryDescription
1.3RequiredNo undefined or critical unspecified behaviour
11.3RequiredNo cast between pointer to object and integer
14.4RequiredControlling expression must be essentially Boolean
17.7RequiredReturn value of non-void function shall be used
21.3RequiredDo not use malloc, calloc, realloc, or free
Static Analysis Tools

PC-lint, Polyspace, Coverity, and cppcheck can automatically check MISRA compliance. Integrate into CI/CD to catch violations before code review.
