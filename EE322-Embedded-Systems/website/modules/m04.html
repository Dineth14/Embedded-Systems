Memory Hierarchy

Principle

The memory hierarchy exploits temporal locality (recently accessed data is likely to be accessed again) and spatial locality (nearby data is likely to be accessed soon). Faster, smaller memories are placed closer to the CPU.

Average Memory Access Time (AMAT):

$$\text{AMAT} = t_{\text{hit}} + \text{miss rate} \times t_{\text{miss penalty}}$$

Cache Operation

Direct-Mapped Cache

Each memory block maps to exactly one cache line: line = (address / block_size) % num_lines.

Simple but prone to conflict misses when two frequently-used addresses map to the same line.

N-Way Set-Associative Cache

Each memory block maps to a set with $N$ ways. The address is checked against all $N$ tags in the set. Reduces conflict misses at the cost of more comparators.

Example

A 4 KB, 4-way set-associative cache with 16-byte lines on a 32-bit system:

  • Total lines: $4096 / 16 = 256$
  • Sets: $256 / 4 = 64$
  • Offset bits: $\log_2(16) = 4$
  • Index bits: $\log_2(64) = 6$
  • Tag bits: $32 - 6 - 4 = 22$

DMA Transfers

Definition

Direct Memory Access (DMA) allows peripherals to transfer data to/from memory without CPU intervention. The CPU programs the DMA controller (source, destination, count, direction) and is free to execute other tasks until the DMA completion interrupt.

DMA modes: memory-to-peripheral, peripheral-to-memory, memory-to-memory. On STM32, each DMA stream can be configured for circular (continuous) or normal (one-shot) mode.
