Insight

CMSIS (Cortex Microcontroller Software Interface Standard) provides a vendor-independent hardware abstraction layer. It includes: core register access functions, SysTick timer interface, NVIC API, and DSP library functions. Using CMSIS improves code portability across Cortex-M devices from different vendors (ST, NXP, TI, Nordic, etc.).

Bare-Metal C Programming

Key Concepts

Bare-metal programming means writing code that runs directly on hardware without an operating system. Critical patterns include:

  • volatile: Tell the compiler a variable may change unexpectedly (hardware registers, ISR-shared variables).
  • Memory-mapped I/O: Peripheral registers accessed at specific addresses via pointers.
  • Struct overlay: Map a C struct over a peripheral register block for readable access.
// Memory-mapped GPIO register access
#define GPIOA_BASE  0x40020000
#define GPIOA_ODR   (*(volatile uint32_t *)(GPIOA_BASE + 0x14))

// Toggle pin 5 (LED on Nucleo boards)
GPIOA_ODR ^= (1 << 5);

Cross-Compilation Toolchain

Embedded development uses a cross-compiler that runs on a host PC but generates code for the target MCU.

# ARM GCC toolchain example
arm-none-eabi-gcc -mcpu=cortex-m4 -mthumb -mfloat-abi=hard \
    -mfpu=fpv4-sp-d16 -Os -Wall -c main.c -o main.o

arm-none-eabi-ld -T linker_script.ld main.o startup.o -o firmware.elf

arm-none-eabi-objcopy -O binary firmware.elf firmware.bin

# Flash to target
st-flash write firmware.bin 0x08000000

Key toolchain components: compiler (gcc), assembler (as), linker (ld), library (newlib-nano), debugger (gdb + OpenOCD/J-Link).
