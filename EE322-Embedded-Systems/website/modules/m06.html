Multitasking

Super-Loop Limitations

The super-loop (while(1)) architecture processes all tasks sequentially. Response time to any event depends on the total loop period. Adding tasks increases worst-case latency for all other tasks.

while (1) {
    read_sensors();      // 2 ms
    process_data();      // 50 ms  ← blocks everything!
    update_display();    // 5 ms
    handle_comm();       // 3 ms
}  // worst-case: 60 ms loop period

Processes & Threads

Thread States

A task transitions: New → Ready → Running → Blocked (waiting for event/mutex/delay) → back to Ready when event occurs → Terminated when deleted. On single-core, only one task is Running at any time.

Race Conditions

Because threads share memory, concurrent access without synchronisation causes race conditions — bugs where output depends on unpredictable timing. Always protect shared resources with mutexes or critical sections.

Context Switching on ARM Cortex-M

The PendSV exception handler performs context switches:

  1. Hardware automatically saves R0–R3, R12, LR, PC, xPSR (32 bytes) onto PSP
  2. PendSV handler saves R4–R11 (32 bytes) via STMDB
  3. Current task's PSP stored in its TCB
  4. Scheduler selects next task
  5. New task's R4–R11 restored via LDMIA
  6. PSP set to new task's stack; BX LR triggers hardware restore

Total overhead: ~80–100 cycles on Cortex-M4 at 168 MHz ≈ 0.5 μs.

Inter-Task Communication

Mutex

Ensures mutual exclusion for shared resources. Supports priority inheritance to prevent unbounded priority inversion.

Semaphore

Binary: Signalling between ISR and task. Counting: Control access to N identical resources.

Message Queue

FIFO buffer for safe data passing. Producer blocks if full; consumer blocks if empty.

Priority Inversion

When a high-priority task H waits for a mutex held by low-priority task L, and medium-priority task M preempts L, H is effectively blocked by M (inversion). Priority inheritance fixes this by temporarily boosting L to H's priority.

FreeRTOS Example

#include "FreeRTOS.h"
#include "task.h"
#include "semphr.h"

SemaphoreHandle_t xMutex;
volatile uint32_t shared_counter = 0;

void vTaskIncrement(void *p) {
    for (;;) {
        xSemaphoreTake(xMutex, portMAX_DELAY);
        shared_counter++;
        xSemaphoreGive(xMutex);
        vTaskDelay(pdMS_TO_TICKS(100));
    }
}

void vTaskDisplay(void *p) {
    for (;;) {
        xSemaphoreTake(xMutex, portMAX_DELAY);
        uint32_t val = shared_counter;
        xSemaphoreGive(xMutex);
        printf("Counter: %lu\n", val);
        vTaskDelay(pdMS_TO_TICKS(500));
    }
}

int main(void) {
    xMutex = xSemaphoreCreateMutex();
    xTaskCreate(vTaskIncrement, "Inc",  256, NULL, 2, NULL);
    xTaskCreate(vTaskDisplay,   "Disp", 256, NULL, 1, NULL);
    vTaskStartScheduler();
    for (;;);
}
